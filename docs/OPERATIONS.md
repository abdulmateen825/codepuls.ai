# Operations Runbook

## Local Verification

PowerShell:

```powershell
.\scripts\test-all.ps1
```

POSIX shell:

```bash
scripts/test-all.sh
```

The full validation checks Docker Compose syntax, FastAPI tests, Spring Boot tests, frontend lint, and frontend production build.

## Smoke Tests

Local services:

```powershell
.\scripts\smoke-test.ps1
```

Production or Nginx-routed stack:

```powershell
.\scripts\smoke-test.ps1 -ThroughNginx
```

POSIX equivalent:

```bash
scripts/smoke-test.sh
scripts/smoke-test.sh nginx
```

## Backups

PostgreSQL is the primary durable store. Reports are stored in PostgreSQL, so database backups include report files.

PowerShell:

```powershell
.\scripts\backup-postgres.ps1
```

POSIX shell:

```bash
scripts/backup-postgres.sh
```

Backups are written to `backups/`, which is ignored by Git.

## Restore

Restores should be done during a maintenance window after taking a fresh backup.

PowerShell:

```powershell
.\scripts\restore-postgres.ps1 -BackupFile .\backups\codepulse-postgres-YYYYMMDD-HHMMSS.sql
```

POSIX shell:

```bash
scripts/restore-postgres.sh backups/codepulse-postgres-YYYYMMDD-HHMMSS.sql
```

Both restore scripts require typing `RESTORE` before continuing.

## Common Incident Checks

- Nginx returns 502: check `backend-core` and `frontend` health.
- Scan stays queued: check `backend-ai` logs and `INTERNAL_API_KEY` consistency.
- Callback unauthorized: verify `INTERNAL_API_KEY` is identical in Spring Boot and FastAPI.
- Findings are missing scanner results: verify scanner binaries are installed in the FastAPI image or configured through path variables.
- Chat or explanation falls back: verify `LLM_PROVIDER` and provider API key settings.

## Data Durability

- PostgreSQL: back up regularly and test restores.
- Qdrant: embeddings can usually be regenerated by rescanning, but volume snapshots reduce recovery time.
- Redis: currently not the source of durable application data.
- Temporary clone workspace: disposable by design.
